# Archived Plans v1 — Local AI Stack

*Архивировано: март 2026*

## Почему архивировано

Первоначальный план предполагал 3 отдельных сервиса + локальный AI-стек:
- **ayda_brain** — отдельный FastAPI-сервис с Ollama + Chroma + DeepSeek
- **tg_gather** — сбор фрагментов + POST в ayda_brain
- **ayda_think** — HTTP-клиент к brain + команды бота

## Почему отказались

1. **Ollama + Chroma + DeepSeek** — сложный локальный стек, медленно на CPU, проблемы с деплоем. Заменяем на OpenAI API + pgvector (уже есть PostgreSQL на Railway).

2. **ayda_brain как отдельный сервис** — overengineering. С облачным OpenAI нет смысла в отдельном сервисе — это просто HTTP-вызовы. Вся логика переезжает в ayda_think.

3. **Общая БД для разных приложений** — не нужно. Каждое приложение хранит свои данные отдельно. Смешивать беговые данные и заметки из Telegram — бессмысленно.

4. **Писать свои gather'ы для всего** — не нужно. Для Instagram/LinkedIn проще ручной импорт архивов. Для браузера есть MCP-коннекторы.

## Что полезного осталось из этих планов

- Концепция Fragment → Chain → Artifact — сохранена
- API-спецификация brain (endpoints, модели) — частично переиспользуется
- Глоссарий (фрагмент, артефакт, цепочка) — сохранён
- Идея источнико-агностичного приёма данных — сохранена (POST /api/fragments)
