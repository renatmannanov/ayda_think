# Step 4 — Пассивная генерация артефактов при /cluster

## Что делаем
После кластеризации (`/cluster`) автоматически генерируем артефакты для крупных кластеров.

## Логика

1. `/cluster` завершает кластеризацию (как сейчас)
2. Для каждого кластера с size ≥ 7:
   - Берём все фрагменты кластера
   - Вызываем `synthesize(cluster_name, fragments)`
   - Сохраняем артефакт с привязкой к cluster_id
3. Прогресс: "Генерирую артефакты... 5/12"

## Ограничения
- Только кластеры ≥ 7 фрагментов (меньше — недостаточно для анализа)
- Лимит: максимум 20 артефактов за прогон (чтобы не тратить слишком много GPT-вызовов)
- Кластеры отсортированы по size DESC → самые крупные в приоритете

## Файлы для изменения
- `services/clustering_service.py` — вызывать synthesis после кластеризации
- Или `bot/brain_handler.py` — вызывать после run_clustering() в cluster_command

## Открытые вопросы
- Делать синхронно (пользователь ждёт) или фоново?
  - На ~20 кластеров × ~3 сек = ~60 сек. Долго для синхронного.
  - Вариант: синхронно кластеризация + имена, потом отдельное сообщение "артефакты готовы через ~1 мин"
  - Или: `/cluster` делает только кластеры, `/cluster --artifacts` добавляет синтез
