# Этап 6 — Артефакты (GPT-синтез)

## Цель
Команда `/artifact [тема]` — GPT анализирует фрагменты по теме, показывает эволюцию мысли, находит связи, предлагает продолжение. Результат сохраняется.

## Ключевые решения
- `/hot` убран — сомнительная ценность на текущем объёме данных
- `/pattern` убран как термин — команда называется `/artifact`, результат = артефакт
- Двухпроходный GPT: 1) выборка релевантных фрагментов, 2) анализ эволюции
- Артефакты не затираются при пересборке кластеров
- Несколько артефактов на кластер (разные по времени, по фокусу)

## Решения по ходу
- **Step 4 отложен в бэклог** (`backlog/passive_artifacts.md`): сначала обкатать /artifact вручную, понять качество промптов, потом автоматизировать
- **Pass 1 (selection) делаем сразу**: 10 кластеров по 25+ фрагментов — без отсева качество анализа падает. Pass 1 отбирает 15-20 ключевых, pass 2 анализирует
- **Кластерный контекст убран**: подтягивание всех фрагментов кластера затапливало релевантные результаты мусором из жирных generic кластеров (500+ фрагментов). Теперь используем только 30 результатов поиска. Кластер — только label в заголовке
