# Approximate cluster assignment

## Проблема
Новые фрагменты (после последней кластеризации) не принадлежат ни одному кластеру.
Полная пересборка кластеров дорогая (UMAP + HDBSCAN на всех фрагментах).

## Решение
Лёгкий approximate-assign: новый фрагмент → cosine similarity к центроидам существующих кластеров → если выше порога, присваиваем.

## Детали
- В `fragment_clusters` добавить поле `assignment_type`:
  - `'clustered'` — назначено HDBSCAN при полной кластеризации
  - `'approximate'` — назначено cosine-assign между кластеризациями
- При полной пересборке (`/cluster`): approximate-записи не удаляются, а переоцениваются HDBSCAN и становятся `'clustered'` (или перемещаются в другой кластер)
- Центроид кластера: среднее эмбеддингов его фрагментов (можно кэшировать в `clusters`)
- Порог: cosine similarity > 0.8 (подобрать экспериментально)

## Когда запускать
- При вставке нового фрагмента (после генерации эмбеддинга)
- Или пакетно (при `/normalize`)

## Зависимости
- Нужно хранить центроиды кластеров (новое поле или отдельная таблица)
